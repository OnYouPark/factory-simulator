# Factory Simulator

가상의 사출 + 후공정 조립 공장을 3D로 시뮬레이션한다. JIPM 기준 OEE 분석 대시보드 포함.

## 라인 구성
사출기 #1 (렌즈) → 표면처리 → 조립
사출기 #2 (베젤) → 조립
사출기 #3 (하우징) → 조립
[전자부품 외부 공급] → 조립 → 출하

## 진행 단계 (MVP)

- [x] Stage 0+1: 셋업 + 공장 외곽
- [x] Stage 2: 장비 배치
- [x] Stage 3: AGV·자재 마커
- [x] Stage 4: 시뮬레이션 엔진 (큐·사이클타임·AGV 연동·시간 가속)
- [x] Stage 5: KPI 계산 + 우측 패널 텍스트 표시
- [x] Stage 6: 대시보드 시각화 (Chart.js 게이지·트렌드)
- [x] Stage 7: 이벤트 트리거 + 시연 시나리오

`python -m http.server 8000` 후 `http://localhost:8000` 접속.

## 시연 시나리오 (5분)

| 구간 | 시간 | 동작 | 보여줄 것 |
|------|------|------|---------|
| 1 | 0:00–0:30 | 라인 구조 정적 화면, 카메라 회전 | 7개 장비 배치, 5개 부품 흐름 |
| 2 | 0:30–1:30 | 시뮬레이션 시작 (30x), AGV 운반 시작 | 각 KPI 값이 채워지는 과정, 메인 게이지·트렌드 차트 활성화 |
| 3 | 1:30–2:30 | 사출 영역 줌인, 표면처리 큐 가득 → 사출기 #1 blocked 등장 | 빨간 깜박임, KPI 카드 상태 배지 동기화 |
| 4 | 2:30–3:30 | "#1 금형" 버튼 클릭 → 90초 강제 정지 | 시간가동률 즉시 하락, 라인 OEE 하향 |
| 5 | 3:30–4:30 | "공급 지연" 버튼 클릭 → 조립 라인 추가 정체 | 라인 전체 OEE 추가 하락, 트렌드 차트의 V자 곡선 |
| 6 | 4:30–5:00 | 리셋 후 마무리 코멘트 | 분석 도구로서의 가치 정리 |

### 핵심 메시지

- **즉시 병목 인식**: 라인 OEE 게이지 + 미니 게이지 색상으로 5초 안에 가장 약한 공정 식별
- **이벤트 영향 시각화**: 금형 교체·공급 지연이 라인 전체 OEE에 미치는 영향을 실시간으로 관찰
- **데이터 기반 의사결정**: 추측이 아닌 OEE 수치로 개선 우선순위 결정

### 시연 팁

- 자동 이벤트가 시연 흐름을 흐릴 수 있다면 `js/events.js`의 `AUTO_EVENT_CONFIG.enabled = false`로 임시 비활성화.
- 속도 슬라이더 30x에서 90초 정지가 약 3초 만에 끝남 — 트렌드 차트의 V자 곡선을 강조하려면 60x 이상 권장.
- 일시정지 중에도 카메라 조작·KPI 마지막 값 유지는 가능 — 특정 지점을 가리키며 설명할 때 사용.
